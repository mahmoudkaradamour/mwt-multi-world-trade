# 🗄 MWT Database Architecture

## Overview

This document describes the database architecture of the Multi World Trade (MWT) platform.

MWT adopts a multi-database strategy designed for:

- Scalability
- Flexibility
- High performance
- Multi-tenant architecture
- Future global expansion

---

# Database Strategy

MWT uses two database technologies:

| Database | Role |
|-----------|-------|
| PostgreSQL | Structured business data |
| MongoDB | Flexible and document-based data |

---

# Primary Database

## PostgreSQL

PostgreSQL is the primary source of truth for the platform.

The following domains are stored in PostgreSQL:

- Users
- Roles
- Permissions
- Stores
- Vendors
- Customers
- Orders
- Payments
- Invoices
- Shipments
- Addresses
- Subscription Plans
- Multi-Tenant Metadata

---

## Why PostgreSQL

PostgreSQL provides:

- ACID compliance
- Transactions
- Foreign key constraints
- High reliability
- Strong data consistency
- Mature ecosystem

# Current Database Schema

## User

Current implementation includes the User model.

```prisma
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  password  String
  createdAt DateTime @default(now())
}
```

---

## User Field Definitions

### id

Unique user identifier.

Type:

```text
UUID
```

Example:

```text
bbe40c98-d792-431f-a953-a98f9eda5ba0
```

---

### email

Unique email address.

Rules:

- Required
- Unique
- Valid email format

---

### password

Stores the bcrypt hashed password.

Important:

```text
Plain text passwords are never stored.
```

---

### createdAt

Automatic creation timestamp.

Generated by:

```text
now()
```

---

# Authentication Storage Flow

```text
User Registration
        │
        ▼
Password Hashing (bcrypt)
        │
        ▼
Prisma ORM
        │
        ▼
PostgreSQL User Table
```

---

# Login Flow

```text
Login Request
      │
      ▼
Find User By Email
      │
      ▼
Compare Password Hash
      │
      ▼
Generate JWT Token
      │
      ▼
Return Access Token
```
